POWER CONNECTIONS:
==================

3.3V Rail (from ESP32):
  ├─ RC522 RFID Module
  ├─ MAX30102 Heart Rate Sensor (IMPORTANT: 3.3V ONLY!)
  └─ LED current limiting resistors (optional)

5V Rail (from USB or external):
  ├─ ESP32 VIN pin
  ├─ MLX90614 Thermal Sensor
  └─ 4-Channel Relay Module VCC

Common Ground:
  ├─ ESP32 GND
  ├─ All sensor modules GND
  ├─ Relay module GND
  ├─ LED cathodes
  ├─ Buzzer (-)
  └─ Raspberry Pi GND (via USB-UART)


I2C BUS SEPARATION (Important!):
=================================

Bus 0 (Wire):  GPIO 21 (SDA), GPIO 22 (SCL) → MLX90614 (5V device)
Bus 1 (Wire1): GPIO 18 (SDA), GPIO 19 (SCL) → MAX30102 (3.3V device)

Note: These buses are separated to avoid voltage conflicts between 5V and 3.3V I2C devices.


RELAY WIRING (Medicine Dispensing + Solenoid Lock):
====================================

Each relay controls one medicine slot or the solenoid lock:

Relay 1 (GPIO 33) → Medicine Slot 1
Relay 2 (GPIO 32) → Medicine Slot 2
Relay 3 (GPIO 25) → Medicine Slot 3
Relay 4 (GPIO 12) → Medicine Slot 4
Relay 5 (GPIO 14) → Solenoid Door Lock (10-second timed unlock)

Relay Module Terminals:
  COM   → Power source (+) for solenoid/motor
  NO    → Solenoid/motor (+)
  NC    → Not used
  GND   → Power source (-)

Active LOW: GPIO HIGH = Relay OFF, GPIO LOW = Relay ON

Solenoid Lock Notes:
  - ESP32 sends `{"command":"unlock_solenoid"}` to activate for 10 seconds
  - Triggered via Admin Panel "Unlock Door" button
  - Use a 12V solenoid bolt lock wired through relay COM/NO on GPIO 14

RGB LED WIRING (Status LED moved to Raspberry Pi — see below):
===============================================================

The RGB status LED is no longer wired to the ESP32.
It is driven by the Raspberry Pi via the medisync-led-status Docker service.

See: Kiosk/MicroControllers/led_service/led_status.py


RASPBERRY PI STATUS LED WIRING:
================================

LED Type: Common Cathode RGB LED

  R → RPi BCM GPIO 17 (via 220Ω resistor)  [configurable: LED_R_PIN]
  G → RPi BCM GPIO 27 (via 220Ω resistor)  [configurable: LED_G_PIN]
  B → RPi BCM GPIO 22 (via 220Ω resistor)  [configurable: LED_B_PIN, not currently driven]
  Common Cathode → GND (Pi physical pin 9 or any GND)

LED States:
  🟢 Slow breathing green  = System idle, backend connected
  🔴 Solid red             = Waiting for finger on heart rate sensor
  🔴→🟢 Fade               = Vitals scan in progress (red=0% → green=100%)
  🟢 Solid green (2s)      = Vitals reading complete
  🔴 Fast blink            = Backend disconnected


RASPBERRY PI RFID STATUS LED WIRING:
======================================

LED Type: Common Cathode RGB LED

  R → RPi BCM GPIO 23 (via 220Ω resistor)  [configurable: RFID_LED_R_PIN]
  G → RPi BCM GPIO 24 (via 220Ω resistor)  [configurable: RFID_LED_G_PIN]
  B → RPi BCM GPIO 25 (via 220Ω resistor)  [configurable: RFID_LED_B_PIN]
  Common Cathode → GND (Pi physical pin 9 or any GND)

LED States:
  🟢 Solid green           = RFID scanner idle, ready to scan
  🔴 Solid red             = Session active (vitals scan in progress)
  🔵 Pulsing blue          = RFID test mode enabled (Admin Panel)

TROUBLESHOOTING TIPS:
=====================

No power:
  - Check USB cable (must support data)
  - Check power LED on ESP32
  - Measure 3.3V and 5V pins with multimeter

Sensor not found:
  - Verify I2C wiring (SDA/SCL not swapped)
  - Check voltage (3.3V vs 5V)
  - Scan I2C bus with scanner sketch
  - Ensure shared GND connections

RFID not reading:
  - Check SPI connections
  - Verify 3.3V power
  - Check antenna connection
  - Test with different card

Relays not clicking:
  - Check GPIO pin numbers
  - Verify relay module power (5V)
  - Test with digitalWrite() sketch
  - Check active LOW vs HIGH

LEDs wrong color:
  - Verify common cathode (not common anode)
  - Check R/G/B pin assignments
  - Test with single color first
  - Add current limiting resistors

No serial communication:
  - Check TX/RX crossover (TX→RX, RX→TX)
  - Verify baud rate (115200)
  - Ensure common ground
  - Close other serial programs
